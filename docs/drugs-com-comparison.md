## 주요 부족한 부분 (우선순위순)

### 1. 임신/수유부 안전 정보 (High Impact)

- **drugs.com**: FDA Pregnancy Category (A/B/C/D/X) + 수유 안전 등급 별도 페이지
- **pillright.com**: ❌ 없음
- **영향**: 임산부/수유부는 약물 정보 검색량이 매우 높은 세그먼트. "임신 타이레놀" 같은 키워드 유입 불가
- **구현 방안**: 식약처 DUR 데이터에 임부금기/수유부금기 정보 이미 포함 → 파싱하여 별도 섹션/서브페이지 생성

### 2. 약물 부작용 전용 페이지 (High Impact)

- **drugs.com**: 각 약물마다 `/side-effects` 서브페이지 (빈도별 분류, 심각도, 사용자 보고)
- **pillright.com**: 상세 페이지 내 `se_qesitm` 필드 하나로 처리. `/drugs/side-effects`는 일반 가이드
- **영향**: "타이레놀 부작용" 검색 키워드 대응 불가
- **구현 방안**: `/drugs/[slug]/side-effects` 서브페이지 SSG → 인덱싱 페이지 2배 확장 (12,000 → 24,000+)

### 3. 질환별 약물 가이드 강화 (High Impact)

- **drugs.com**: `/condition/` — 질환별 치료약 목록 + 사용자 평점 랭킹
- **pillright.com**: `/drugs/conditions` 존재하지만 정적 데이터 기반
- **영향**: "고혈압 약", "당뇨약 종류" 등 롱테일 키워드 기회
- **구현 방안**: 약물-질환 매핑 테이블 + 질환별 SSG 페이지 생성

### 4. 약물 뉴스/회수/안전 알림 (Medium Impact)

- **drugs.com**: FDA 알림, 약물 회수 정보, 신약 승인 뉴스
- **pillright.com**: `/news` 페이지 존재하지만 콘텐츠 소스 미연결
- **영향**: 식약처 공지사항/회수 정보 자동 수집하면 뉴스성 트래픽 확보 가능
- **구현 방안**: 식약처 의약품 회수 API + 안전성 정보 RSS 크롤링 → `/news` 자동 업데이트

### 5. 복용량 가이드 강화 (Medium Impact)

- **drugs.com**: 성인/소아/연령별/체중별 복용량 + 놓친 복용 시 대처법
- **pillright.com**: `use_method_qesitm` 텍스트만 표시
- **영향**: "타이레놀 어린이 복용량" 같은 실용적 검색 키워드
- **구현 방안**: 용법용량 텍스트 구조화 파싱 (성인/소아 분리, 테이블 형태 렌더링)

### 6. 약물 가격 비교 (Medium Impact — 한국 특화)

- **drugs.com**: GoodRx 연동 가격 비교
- **pillright.com**: ❌ 없음
- **영향**: 건강보험심사평가원 약가 데이터 활용 가능. "처방약 가격" 검색 유입
- **구현 방안**: 심평원 약가 공개 API 연동 → 약물 상세 페이지에 약가 섹션 추가

### 7. Q&A / 커뮤니티 (Low-Medium Impact)

- **drugs.com**: Answers 섹션 — 사용자 질문 + 약사 답변 (UGC → SEO 금광)
- **pillright.com**: ❌ 없음
- **영향**: UGC 콘텐츠는 롱테일 SEO에 매우 강력하나 운영 리소스 필요
- **구현 방안**: Phase 2 이후 검토. 초기에는 FAQ 확장으로 대체

### 8. 인쇄용 약물 정보 (Low Impact)

- **drugs.com**: Print-friendly 약물 가이드 PDF
- **pillright.com**: ❌ 없음
- **영향**: 의료진/약사가 환자에게 제공하는 용도
- **구현 방안**: CSS `@media print` + 인쇄 버튼 추가

| 개선 | 노력 | SEO 효과 | 비고 |
|------|------|----------|------|
| 약물별 `/drugs/[slug]/side-effects` 서브페이지 | 중 | 높음 | 인덱싱 페이지 2배 |
| 임신/수유 안전 정보 섹션 추가 | 중 | 높음 | DUR 데이터 활용 |
| 질환별 페이지에 실제 약물 연결 | 중 | 높음 | 롱테일 키워드 |
| 식약처 회수/안전성 정보 자동 수집 | 중 | 중 | 뉴스성 트래픽 |
| 각 약물 FAQ 확장 (구조화 데이터) | 소 | 중 | FAQ 리치 스니펫 |
| 약가 정보 (심평원 API) | 대 | 중 | 실용 정보 |

## 요약

pillright.com은 **상호작용 체크 + 4종 아이템(약/영양제/식품/한약재)** 조합에서 drugs.com보다 우위에 있음.
하지만 **개별 약물의 정보 깊이**(부작용 서브페이지, 임신 안전, 복용량 가이드, 질환-약물 매핑)에서 차이가 큼.

### ROI 상위 3가지

1. **임신/수유 안전 정보** — 검색량 높고, 식약처 DUR 데이터에 이미 포함
2. **부작용 서브페이지** — 약물별 URL 생성으로 인덱싱 페이지 2배 확장
3. **질환-약물 연결 강화** — 롱테일 키워드 대응
