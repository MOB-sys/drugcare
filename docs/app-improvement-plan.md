# Flutter 앱 개선 검토 — 웹 기능 기반 분석

> 작성일: 2026-03-11
> 기준: 서비스 분석(service-analysis.md) + 웹/앱 코드 리서치 + 백엔드 API 현황

---

## 현황 요약

### 앱 현재 상태
- **화면:** 9개 (홈, 검색, 결과, 복약함, 리마인더, 리마인더폼, 설정, 법률, 피드백)
- **API 호출:** 6개 서비스 (Drug, Supplement, Interaction, Cabinet, Reminder, Feedback)
- **테스트:** 131개 통과
- **광고:** AdMob 3종 (배너, 전면, 네이티브)

### 백엔드 API에 있지만 앱에서 안 쓰는 엔드포인트

| 엔드포인트 | 용도 | 우선순위 |
|-----------|------|---------|
| `GET /drugs/side-effects/search` | 부작용 역검색 | ★★★ |
| `GET /drugs/conditions/search` | 질환별 주의약물 | ★★★ |
| `GET /drugs/symptoms/search` | 증상별 약물 검색 | ★★★ |
| `GET /drugs/identify` | 알약 식별 (색상/모양/각인) | ★★☆ |
| `GET /drugs/recent` | 최근 등록 약물 | ★★☆ |
| `GET /reviews/*` | 사용자 리뷰/별점 CRUD | ★★★ |
| `GET /reviews/summary` | 리뷰 통계 | ★★★ |
| `POST /reviews/{id}/helpful` | 리뷰 도움됨 | ★★☆ |
| `GET /drugs/browse` | 가나다 브라우징 | ★☆☆ |
| `GET /drugs/browse/counts` | 초성별 카운트 | ★☆☆ |
| `GET /supplements/browse` | 영양제 브라우징 | ★☆☆ |
| `GET /drugs/count` | 전체 약물 수 | ★☆☆ |
| `GET /supplements/count` | 전체 영양제 수 | ★☆☆ |
| `POST /metrics/event` | 이벤트 분석 | ★★☆ |

## 개선 항목 검토

### 우선순위 기준
- **수익 기여**: 사용자 체류 시간 → 광고 노출 → 수익
- **리텐션 기여**: 매일 들어올 이유를 만드는가
- **개발 난이도**: 기존 API 있으면 프론트만 추가
- **경쟁 대응**: 필라이즈 등 대비 차별화

---

### P0 — 필수 (앱 가치 대폭 상승)

#### 1. 약물/영양제 상세 정보 강화
**현재:** 앱에서 `GET /drugs/{id}` 호출하지만 상세 화면이 없음 (검색→선택→상호작용 체크만)
**추가할 것:**

| 항목 | 설명 | API | 난이도 |
|------|------|-----|--------|
| DUR 안전성 정보 | 임부/고령자/용량 제한/기간 제한/중복 성분 | `DrugDetail.dur_safety[]` (이미 응답에 포함) | 중 |
| 식품-약물 상호작용 | 과일, 유제품, 알코올, 카페인 등 | `DrugDetail.intrc_qesitm` 파싱 | 중 |
| 성분 테이블 | 성분명, 함량, 단위 | `DrugDetail.ingredients[]` | 하 |
| 효능/용법/부작용 | 상세 텍스트 섹션 | `DrugDetail.*_qesitm` | 하 |
| 보관 방법 | 저장 조건 | `DrugDetail.deposit_method_qesitm` | 하 |

**구현 방향:**
- 새 화면: `DrugDetailScreen` + `SupplementDetailScreen`
- 라우트: `/drugs/:id`, `/supplements/:id`
- 검색 결과 카드 탭 → 상세 화면 이동 (현재는 선택 토글만)
- 상세 화면 내 "상호작용 체크" CTA 버튼
- 상세 화면 내 "복약함 추가" 버튼
- **광고 배치:** 상세 화면 하단 배너 광고 (체류 시간 증가 → 수익)

**예상 작업량:** 화면 2개 + 위젯 5~6개 + 모델 보강 = 2~3일

---

#### 2. 사용자 리뷰 시스템
**현재:** 백엔드 Review API 완비, 웹에서 사용 중, 앱에는 없음
**추가할 것:**

| 기능 | API | 화면 |
|------|-----|------|
| 별점 + 리뷰 작성 | `POST /reviews/` | 상세 화면 내 |
| 리뷰 목록 (페이징) | `GET /reviews/{type}/{id}` | 상세 화면 내 |
| 평균 별점/통계 | `GET /reviews/{type}/{id}/summary` | 상세 화면 헤더 |
| 도움됨 버튼 | `POST /reviews/{id}/helpful` | 리뷰 카드 |
| 내 리뷰 삭제 | `DELETE /reviews/{id}` | 리뷰 카드 |

**구현 방향:**
- 상세 화면 하단 섹션으로 통합 (별도 화면 불필요)
- `ReviewService` + `reviewProvider` 추가
- 별점 위젯 (StarRating) 재사용
- 리뷰 작성 시 디바이스 ID 기반 (1인 1리뷰, 수정 가능)
- **리텐션 효과:** 내가 쓴 리뷰 → 다시 확인하러 방문

**예상 작업량:** 서비스 1개 + 프로바이더 1개 + 위젯 4개 = 1~2일

---

#### 3. 증상별 약물 검색
**현재:** 없음 (검색은 이름 기반만)
**추가할 것:** "두통이 있어요" → 관련 약물 목록

| 기능 | API |
|------|-----|
| 증상 키워드 검색 | `GET /drugs/symptoms/search?q=두통` |
| 자주 찾는 증상 칩 | 로컬 데이터 (두통, 소화불량, 감기, 생리통 등) |

**구현 방향:**
- 홈 화면에 "증상으로 약 찾기" 카드 추가
- 새 화면: `SymptomSearchScreen`
- 상단: 자주 찾는 증상 칩 (8~10개)
- 하단: 검색 결과 리스트 → 탭 시 상세 화면
- **사용자 가치:** "어떤 약 먹어야 하지?" 시나리오 해결
- **광고 배치:** 결과 리스트 중간 네이티브 광고

**예상 작업량:** 화면 1개 + 서비스 메서드 1개 + 위젯 2개 = 1일

### P1 — 높음 (사용성 + 체류시간 향상)

#### 4. 부작용 역검색
**현재:** 없음
**추가할 것:** "어지러움이 있어요" → 원인 가능성 있는 약물 목록

| 기능 | API |
|------|-----|
| 부작용 키워드 검색 | `GET /drugs/side-effects/search?q=어지럼` |
| 자주 찾는 부작용 칩 | 로컬 데이터 (어지럼, 졸음, 구역질, 발진 등) |

**구현 방향:**
- 홈 화면에 "부작용 확인" 카드 추가
- 새 화면: `SideEffectSearchScreen` (증상 검색과 유사 레이아웃)
- 키워드 하이라이팅 (부작용 텍스트에서 검색어 강조)
- **사용자 가치:** "이 증상이 약 때문인가?" 불안 해소

**예상 작업량:** 화면 1개 + 서비스 메서드 1개 = 1일

#### 5. 질환별 주의약물
**현재:** 없음
**추가할 것:** "당뇨인데 조심할 약은?" → 관련 주의 약물 목록

| 기능 | API |
|------|-----|
| 질환 키워드 검색 | `GET /drugs/conditions/search?q=당뇨` |
| 질환 카드 목록 | 로컬 데이터 (당뇨, 고혈압, 임산부, 간질환 등 12종) |

**구현 방향:**
- 새 화면: `ConditionSearchScreen`
- 카드 그리드 (웹과 동일한 12종 질환 카테고리)
- 카드 탭 → 해당 질환 주의 약물 리스트
- **사용자 가치:** 기저질환 있는 사용자 (핵심 타겟 40~60대)에게 매우 유용

**예상 작업량:** 화면 1개 + 서비스 메서드 1개 = 1일

#### 6. 공유 기능
**현재:** 없음 (상호작용 결과를 공유할 수 없음)
**추가할 것:**

| 기능 | 구현 방식 |
|------|----------|
| 결과 공유 | `share_plus` 패키지 → 네이티브 공유 시트 |
| 카카오톡 공유 | `kakao_flutter_sdk_share` → 카카오 링크 |
| 결과 스크린샷 | `screenshot` 패키지 → 이미지로 저장/공유 |

**구현 방향:**
- 결과 화면 AppBar에 공유 아이콘 추가
- 공유 시 웹 URL 포함: `https://pillright.com/check/result?items=...`
- 카카오톡 공유: 결과 요약 + OG 이미지 (웹 URL)
- **웹→앱 시너지:** 공유받은 사람이 웹으로 유입 → 앱 설치 유도

**예상 작업량:** 1일 (share_plus 기본) ~ 2일 (카카오 포함)

#### 7. 이벤트 분석 (Metrics)
**현재:** `POST /metrics/event` API 있지만 앱에서 호출 안 함
**추가할 것:**

| 이벤트 | 시점 |
|--------|------|
| `app_open` | 앱 시작 |
| `interaction_check` | 상호작용 체크 실행 |
| `search` | 검색 실행 |
| `cabinet_add` | 복약함 추가 |
| `detail_view` | 약물/영양제 상세 조회 |
| `review_submit` | 리뷰 작성 |
| `share` | 결과 공유 |

**구현 방향:**
- `MetricsService` + 글로벌 헬퍼 (`trackEvent("app_open")`)
- 앱 라이프사이클 감지 (`WidgetsBindingObserver`)
- 비동기 fire-and-forget (실패 시 무시)
- **KPI 측정 필수:** DAU, 체크 횟수, 리텐션 등

**예상 작업량:** 서비스 1개 + 각 화면에 호출 추가 = 0.5일

### P2 — 보통 (기능 확장)

#### 8. 알약 식별 (텍스트 기반)
**현재:** 없음
**추가할 것:** 색상/모양/각인 텍스트로 알약 찾기

| 기능 | API |
|------|-----|
| 색상 선택 (10+색) | 로컬 UI |
| 모양 선택 (7+종) | 로컬 UI |
| 각인 텍스트 입력 | 사용자 입력 |
| 검색 | `GET /drugs/identify?color=하양&shape=원형&imprint=TY` |

**구현 방향:**
- 새 화면: `PillIdentifyScreen`
- 3단계 필터: 색상 칩 → 모양 칩 → 각인 입력 → 검색
- 결과: 약물 이미지 + 이름 → 탭 시 상세
- Phase 2에서 카메라 촬영 → AI 식별로 확장 가능
- **주의:** 카메라 기능은 별도 (image_picker + ML 필요), 이번엔 텍스트만

**예상 작업량:** 화면 1개 + 서비스 메서드 1개 = 1~2일

---

#### 9. 최근 등록 약물 / 안전 뉴스
**현재:** 없음
**추가할 것:** 홈 화면에 "최근 등록 약물" 섹션

| 기능 | API |
|------|-----|
| 최근 30일 등록 약물 | `GET /drugs/recent?days=30&limit=5` |

**구현 방향:**
- 홈 화면에 수평 스크롤 카드 리스트
- 탭 시 상세 화면 이동
- 주기적 갱신 → "새로운 약 정보" → 재방문 동기
- **리텐션:** 새 정보가 있다 → 다시 열어볼 이유

**예상 작업량:** 서비스 메서드 1개 + 홈 위젯 1개 = 0.5일

**현재:** 퀵서치바 + 복약함 요약 + 건강팁 + 네이티브 광고 (4개 카드)

**예상 작업량:** 위젯 2개 + 레이아웃 조정 = 1일

### P3 — 낮음 (장기 고려)

#### 11. 가나다 브라우징
- 웹에서는 SEO 목적이지만, 앱에서는 검색이 더 빠름
- 앱에서 우선순위 낮음 → **스킵 권장**

#### 12. 약물 비교
- 2개 약물 나란히 비교 (웹에만 있음)
- 앱에서 구현하면 화면이 좁아 UX 어려움 → **태블릿용으로만 고려**

#### 13. 전문가 모드
- 의료 전문가용 밀집 정보 화면
- 앱 타겟(일반 소비자)과 맞지 않음 → **스킵**

#### 14. 카테고리/성분 가이드
- 웹에서는 SEO 콘텐츠 역할
- 앱에서는 읽기 전용 → 웹뷰로 연결이 더 효율적
- **WebView로 pillright.com/categories, /ingredients 연결 권장**

## 구현 로드맵

### Phase A (1주) — 핵심 상세 + 리뷰

| 일차 | 작업 | 산출물 |
|------|------|--------|
| 1일 | DrugDetailScreen + SupplementDetailScreen 기본 | 화면 2개 |
| 2일 | DUR 안전성 + 식품 상호작용 + 성분 테이블 위젯 | 위젯 5개 |
| 3일 | ReviewService + 리뷰 목록/작성/별점 위젯 | 서비스 1 + 위젯 4 |
| 4일 | 테스트 + 라우팅 연결 + 검색 결과→상세 네비게이션 | 테스트 20+ |
| 5일 | 광고 배치 (상세 화면 배너) + QA | 광고 1 슬롯 |

### Phase B (3~4일) — 탐색 도구

| 일차 | 작업 | 산출물 |
|------|------|--------|
| 1일 | SymptomSearchScreen + SideEffectSearchScreen | 화면 2개 |
| 2일 | ConditionSearchScreen + 질환 카드 데이터 | 화면 1개 |
| 3일 | 홈 화면 리뉴얼 (도구 모음 + 최근 약물) | 위젯 2개 |
| 4일 | 테스트 + QA | 테스트 15+ |

### Phase C (2일) — 공유 + 분석

| 일차 | 작업 | 산출물 |
|------|------|--------|
| 1일 | share_plus 연동 + 결과 공유 + 카카오 공유 | 공유 기능 |
| 2일 | MetricsService + 이벤트 트래킹 전 화면 | 분석 기능 |

### Phase D (2일) — 알약 식별 + 마무리

| 일차 | 작업 | 산출물 |
|------|------|--------|
| 1일 | PillIdentifyScreen (텍스트 기반) | 화면 1개 |
| 2일 | 전체 통합 테스트 + 빌드 + 스토어 제출 | 릴리스 |

### 수익 효과
| 항목 | 현재 | 개선 후 | 근거 |
|------|------|--------|------|
| 평균 체류 시간 | ~1분 (체크만) | ~3분 (상세+리뷰+탐색) | 상세 페이지 3배 체류 |

| DAU → 광고 수익 | 기준 1x | 2~3x | 체류시간 × 노출 수 |
